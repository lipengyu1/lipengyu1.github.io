    # 泛型

    使用传统方法的问题分析

    1. 不能对加入到集合ArrayList中的数据进行约束（不安全）
    2. 遍历的时候，需要进行类型转换，如果集合中的数据量较大，对效率有影响
    public class Generic01 {
    public static void main(String[] args) {
        //使用传统方式解决
        ArrayList list = new ArrayList();
        list.add(new Dog(10, "一"));
        list.add(new Dog(11, "二"));
        list.add(new Dog(12, "三"));

        list.add(new Cat(99, "四"));//ClassCastException
        //遍历
        for (Object o : list) {
            Dog dog = (Dog) o;//向下转型
            System.out.println(((Dog) o).getAge() + ((Dog) o).getName());
        }
    }
}

    # 泛型的好处

    1. 编译时，检查添加元素的类型，提高了安全性
    2. 减少了类型转化次数，提高了效率
    3. 再提示编译警告

    # 泛型介绍

    1. 泛型又称参数化类型，解决数据类型的安全性问题
    2. 在声明或实例化时只要指定好需要的具体类型即可
    3. Java泛型可以保证程序在编译时没有发出警告，在运行时就不会产生ClassCastException 异常
    4. 泛型的作用时，可以在类型声明通过一个标识表示类中的某个属性，或者是某个方法的返回值的类型，或者是参数类型
    public class Generic03 {
    public static void main(String[] args) {
        Person<String> stringPerson = new Person<String>("一");
        stringPerson.show();//class java.lang.String
        /*
        可理解为
        class Person{
            String s;
            public Person(String s) {
                this.s = s;
            }
            public String f(){
                return s;
            }
        }
         */
        Person<Integer> integerPerson = new Person<Integer>(100);
        integerPerson.show();//class java.lang.Integer
    }
}
//可以在类型声明通过一个标识表示类中的某个属性，或者是某个方法的返回值的类型，或者是参数类型
class Person<E>{
    E s;//E 表示s的数据类型，该数据类型在定义Person对象的时候指定，即在编译期间，就确定E是什么类型
    public Person(E s) {//E也可以是参数类型
        this.s = s;
    }
    public E f(){//返回类型
        return s;
    }
    public void show(){
        System.out.println(s.getClass());
    }
}

    # 自定义泛型方法

    # 基本语法

    修饰符<T,R>返回类型 方法名（参数列表）{
    }

    # 细节

    1. 泛型方法可以定义在普通类中，。也可以定义在泛型类中
    2. 当泛型方法被调用时，类型会确定
    3. Public void eat（E e）{}，修饰符后没有<T,R..> eat 方法不是泛型方法，而是使用了泛型
    public class CustomMethodGenric {
    public static void main(String[] args) {
        Car car = new Car();
        car.fly("一",100);//当调用方法时，传入参数，编译器就会确定类型
        //class java.lang.String
        //class java.lang.Integer
        car.fly(300,100.1);//当调用方法时，传入参数，编译器就会确定类型
        //class java.lang.Integer
        //class java.lang.Double

        //测试
        //T->String,R->ArrayList
        Fish<String, ArrayList> stringArrayListFish = new Fish<>();
        stringArrayListFish.hello(new ArrayList(),11.5f);
        //class java.util.ArrayList
        //class java.lang.Float
    }
}
//1.泛型方法可以定义在普通类中，。也可以定义在泛型类中
//2.当泛型方法被调用时，类型会确定
//3.Public void eat（E e）{}，修饰符后没有<T,R..> eat 方法不是泛型方法，而是使用了泛型
class Car{//普通类
    public void run(){//普通方法
    }
    //1.<T,R>就是泛型
    //2.提供给fly使用的
    public<T,R> void fly(T t,R r){//泛型方法，运行时必须指定相应的类型
        System.out.println(t.getClass());//1.Stirng["一"]  2.Integer(自动装箱)[300]
        System.out.println(r.getClass());//1.Integet(自动装箱)[100]  2.Double(自动装箱)[100.1]
    }
}
class Fish<T,R>{//泛型类
    public void run(){//普通方法
    }
    public<U,M> void eat(U u,M m){//泛型方法
    }

    //说明
    //1.hi（）方法不是泛型方法
    //2.是hi（）方法使用了类声明的 泛型
    public void hi(T t){
    }
    //泛型方法，可以使用类声明的泛型，也可以自己声明泛型
    public<K> void hello(R r,K k){//R为泛型类的R，K是该泛型方法的K
        System.out.println(r.getClass());//ArrayList
        System.out.println(k.getClass());//Float
    }
}

    # 泛型的继承与通配符

    1. 泛型不具备继承性
    List<Oject> list = new ArrayList<String>()//错误，泛型不具备继承
    1
    1. <?>:支持任意泛型
    2. <? extends A>:支持A类以及A类的子类，规定了泛型的上限
    3. <? super A>:支持A类以及A的父类，不限于直接父类，规定了泛型的下限 
    public class GenericExtends {
    public static void main(String[] args) {
        //举例说明下列三个方法的使用
        List<Object> list1 = new ArrayList<>();
        List<String> list2 = new ArrayList<>();
        List<AA> list3 = new ArrayList<>();
        List<BB> list4 = new ArrayList<>();
        List<CC> list5 = new ArrayList<>();
        
        //如果是List<?> c,可以接收任意的泛型类型
        printCollection1(list1);
        printCollection1(list1);
        printCollection1(list1);
        printCollection1(list1);
        printCollection1(list1);
        //如果是List<? extends AA> c,可以接收AA 或者AA 的子类
        printCollection2(list3);//AA类
        printCollection2(list4);//BB类
        printCollection2(list5);//CC类
        //如果是List<? super AA> c，支持AA类以及AA类的父类，不限于直接父类
        printCollection3(list1);//Object类
        printCollection3(list3);//AA类
    }
    //说明：List<?> 表示任意的泛型类型都可以接收
    public static void printCollection1(List<?> c){
        for (Object object : c) {//通配符，取出时，就是Object
            System.out.println(object);
        }
    }
    // ? extends AA 表示 上限，可以接收AA 或者AA 的子类
    public static void printCollection2(List<? extends AA> c){
        for (Object object : c) {
            System.out.println(object);
        }
    }
    //? super 子类类名AA，支持AA类以及AA类的父类，不限于直接父类
    //规定了泛型的下限
    public static void printCollection3(List<? super AA> c){
        for (Object object : c) {
            System.out.println(object);
        }
    }

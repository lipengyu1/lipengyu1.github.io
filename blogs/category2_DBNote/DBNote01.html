    数据库系统概论与关系化数据库设计

    lpy 3/21/2022 数据库概论

    # 数据库概论

    # 数据库

    1. 存放数据的仓库
    2. 按一定数据模型组织和存储
    3. 需要频繁搜索和读取

    # 数据库管理系统

    管理数据库的软件

    # 关系化数据库管理系统

    1. 基于关系化数据模型的数据库管理系统
    2. 数据存储在很多表中

    # 关系化数据库的设计

    1. 了解需求
    2. 抽象出实体和实体间的联系，使用ER图来描述
    3. 把ER图转化为关系（表）
    4. 将关系分开以避免冗余

    # Mysql

    数据库->数据表->记录

    1. mysql -u用户名 -p密码； //登陆
    2. net start mysql； //启动数据库
    3. net stop mysql； //停止数据库
    4. show databases； //列出当前系统的数据库列表
    5. use 数据库名； //打开数据库
    6. show tables; //展示数据库中的表
    7. select * from 表名； select * from 表明 \G; 查看表中的所有数据

    DB 数据库 DBA 数据库管理员 DBMS 数据库管理系统 DBS 数据库系统

    # 需求分析

    实体：表示数据库中描述显示世界中的对象，，实体可也是人也可以是物体实物，也可以是抽象的概念

    关系类型：一对一、一对多、多对多

    电子商务网站：

    用户：用户ID 用户名 密码 电话 邮箱 地址
    商品：商品ID 商品名称 商品描述 单价 单位
    订单：订单ID 订单价格 订单状态
    供应商：供应商ID 供应商名称 联系人 电话 地址

    # E-R转化表：逻辑模型转化物理模型

    • 实体类型的转换
    1. 实体类型的转换
    2. 将每个实体类型转换成一个关系模式。
    3. 实体的属性即为关系模式的属性。
    4. 实体标识符即为关系模式的键。
    • 二元联系类型的转换
    1. 实体集转化成实体表，实体属性转成关系表字段，主属性转成表主键
    2. M-N多对多关系：生成关系表，关系表的主键为联合主键，由两个实体表的主键构成
    3. 1-N一对多的关系：不生成关系表，在多端表中加入单端表的主属性
    4. 1-1关系：不生成关系表，选择两表的任意一表加入另一表的主键即可

    # 范式

    范式是符合某一种级别的关系模式的集合
    关系数据库中的关系必须满足一定的要求，满足不同程度要求的为不同范式
    范式的种类： 1NF,2NF,3NF,BCNF

    范式的目的：消除冗余

    不合理表设计：数据冗余、操作异常（插入异常、更新异常、删除异常）

    部分函数依赖：决定因⼦是多元素的，但是单因⼦也能决定C

    传递依赖：学号、姓名、系号、系名称、系位置

    • 1NF第一范式：表中没有多值属性

    • 2NF 第二范式：不存在 非关键字段对任一候选关键字段的部分函数依赖

    • 3NF 第三范式：没有非关键字段传递依赖主键，非关键字段只依赖于主键

    违反第二范式：存在非关键字段对主键某一字段（学号、年龄）的部分函数依赖
    拆表：将相关的信息放在一张表中，不存在部分函数依赖

    违反第三范式：存在非关键字段对主键的传递依赖
    拆表：满足第三范式,非关键字段都是由主键决定
